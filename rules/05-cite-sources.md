# 规则 05 — 引用必须可追溯

## 原则

任何陈述都必须能被读者**当场核对**。模糊指代、记忆型引用、未署来源的"通常做法"都被视为不可接受。

## 引用格式

| 场景 | 推荐格式 | 例子 |
|------|---------|------|
| 单行 | `path/to/file.ext:LINE` | `src/auth.py:142` |
| 多行区段 | `path/to/file.ext:START-END` | `src/auth.py:142-156` |
| VS Code 扩展中（可点击） | `[file.ext:LINE](path/to/file.ext#LLINE)` | `[auth.py:142](src/auth.py#L142)` |
| 命令输出 | 三反引号代码块 + 命令本身在前 | ` ``` $ pytest tests/test_auth.py::test_lock ... ``` ` |
| 论文 / 文档 | `<title or DOI> §<section> p.<page>` | `Smith 2024 §3.2 p.7` |
| Git | 完整 commit hash（前 7 位起） | `commit abc1234` |

## 必须做（MUST）

- 每一条对仓库内容的断言 → 附 `file:line`。
- 每一条对外部资源的断言 → 附 URL / DOI / 章节号。
- 每一条对运行时行为的断言 → 附实际的命令与输出。
- PR 描述、commit message、回复用户：所有事实性陈述都符合上述要求。

## 禁止做（MUST NOT）

- ❌ "在 auth 模块某处" → 应给出具体 `auth.py:142` 或 `auth/login.py:88`。
- ❌ "记得文档里说过" → 找到文档、给出链接 + 章节。
- ❌ "我之前修过类似的 bug" → 找到那个 commit，给出 hash。
- ❌ 在论文中粘贴一个数字而不能指出它来自哪个脚本/CSV/参考文献。

## 何时可以省略引用

只有以下情况可以不显式引用：

- 通用计算机科学常识（如"单链表插入是 O(1)"）；
- 当前对话中刚刚由用户提供的事实（且未跨越上下文压缩）；
- 完全是 agent 自己刚做的修改，且写在同一句中（"我在 `src/auth.py:142` 加了锁"）。

## 自检触发器

下列任一情况出现，agent 应主动自检本规则：

- 即将给出一个不带 `file:line` 的代码位置陈述；
- 即将复述文档/论文/issue 内容而未给链接；
- 即将复述命令输出而未在同一回合实际运行该命令；
- 在论文/报告中即将贴入数字、日期、系数而未当场重新读取来源（参见规则 01 的"写论文条款"）。
